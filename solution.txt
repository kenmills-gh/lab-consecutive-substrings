1. Initialize an empty array called substrings to store the results.
2. Start an outer loop with an index i from 0 up to the length of the input string. This loop tracks the starting character of each new set of substrings.
3. Inside the outer loop, initialize an empty string variable called currentSubstring.
4. Start an inner loop with an index j starting at i and going up to the length of the input string. This loop tracks the ending character.
5. Inside the inner loop, append the character at string[j] to currentSubstring.
6. Push the current value of currentSubstring into the substrings array.
7. Once both loops have fully executed, return the substrings array.

And a written explanation of your solution:

The type of algorithm chosen for this problem is an Iterative Brute Force algorithm.

How it works:

The solution treats the string like an array of characters. It uses two nested loops. 
- The outer loop iterates through the string to set the starting position of the substring.
- The inner loop begins at that starting position and iterates to the end of the string, cumulatively adding one character at a time to a temporary string variable. 
- After each character is added, the new substring is pushed to the results array.

------------------------------------------------------------------------------------------------------------------------------------------------------------------------
Time Complexity:

The scalable element in this code is 'n', which represents the length of the input string.

- Loop Iterations: The outer loop runs 'n' times. For every iteration of the outer loop, the inner loop runs 'n - i' times. This results in roughly (n * n) / 2 total loop iterations, which simplifies to O(n^2) iterations.
- Operations inside the loop: Inside the inner loop, we perform string concatenation. In many environments, creating a new string by appending a character takes time proportional to the length of the new string. 
- Total Time Complexity: Because we are generating O(n^2) substrings, and the average length of these substrings scales with 'n', the strict worst-case Time Complexity is O(n^3).

Space Complexity:

The space complexity is also O(n^3). We are storing (n^2 + n) / 2 total substrings in an array, and the sum of the lengths of all these generated substrings scales cubically as the input string grows.
